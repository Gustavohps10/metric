# ADR-004 — Arquitetura de Plugins PRO via Module Runtime com Distribuição Privada e Licenciamento Dinâmico

## Status

**Proposto**

## Data

12 de Abril de 2026

## Autor

Gustavo Henrique Pereira dos Santos

---

## 1. Contexto

O sistema Timelapse evolui para uma arquitetura _local-first_ com extensibilidade modular, permitindo a separação entre:

- **Funcionalidades FREE:** Core público.
- **Funcionalidades PRO:** Módulos privados.
- **Futuras extensões:** Marketplace.

### 1.1 Separação de domínio entre CORE e PRO

As funcionalidades PRO não podem estar presentes no bundle público por motivos de segurança de licenciamento, controle de distribuição, proteção de IP e monetização via assinatura.

### 1.2 Necessidade de extensibilidade em runtime

O sistema precisa permitir a injeção de módulos após validação de licença, carregamento dinâmico de funcionalidades, atualização independente de features PRO e isolamento entre CORE e PRO.

### 1.3 Restrição de bundling

O modelo atual de build único do Electron não pode incluir código PRO estático, pois o código estaria acessível no bundle final, facilitando o bypass de licença e violando o modelo de monetização.

### 1.4 Distribuição privada de módulos

Os módulos PRO serão desenvolvidos em um repositório privado contendo analytics avançado, insights de produtividade e integrações enterprise. Esses módulos dependem do SDK público fornecido pelo CORE.

---

## 2. Decisão

Adota-se uma arquitetura baseada em **Module Runtime System** com carregamento dinâmico de módulos PRO sob autorização de licença.

### 2.1 Separação de responsabilidades

- **CORE (Público):** Responsável pela engine de execução, UI base, SDK de módulos e controle de slots extensíveis. Não contém lógica PRO.
- **PRO (Privado):** Responsável por módulos pagos, analytics e IPC channels estendidos. Importa o SDK do CORE e implementa módulos compatíveis.

### 2.2 Mecanismo de ativação de PRO

O CORE conterá apenas slots de extensão (pontos de resolução), sem lógica interna:

```javascript
if (license.isPro) {
  await moduleLoader.load('analytics')
}
```
